☁️ Google 雲端硬碟同步 (像 Google Sheet 一樣自動雙向存取)

將所有的租屋資料（rental_helper_db.json）儲存在您自有的 Google Drive 雲端硬碟中，安全隱私不外洩給第三方。 任何一台裝置（包含家人的手機）只要登入同一個 Google 帳號，或是由擁有者將雲端硬碟中的該檔案分享給家人的 Google 帳號，就能實現多人共同編輯、自動即時同步！

📋 5 分鐘快速設定教學（100% 個人資料隱私自主）：
  1. 前往 Google Cloud 主控台 並登入您的 Google 帳號（例如 youngyang@17.media）。
  2. 建立新專案（例如命名為 Rental Helper）。
  3. 在左側選單選擇「API 和服務」>「OAuth 同意畫面 (OAuth consent screen)」：
    • User Type 選擇「外部 (External)」（若使用 Workspace 帳號亦可選內部）。
    • 填寫應用程式名稱（例如 楊爸爸房租助手）與您的聯絡信箱。
    • 在「範圍 (Scopes)」步驟中，點選「新增或移除範圍」，手動在底部貼入並新增：
      https://www.googleapis.com/auth/drive
      https://www.googleapis.com/auth/userinfo.email
      （⚠️ 重要：請使用此完整雲端權限。由於 Google 安全限制，預設的 drive.file 權限不允許本程式讀寫他人「共用給您」的檔案。升級此權限後，家人才可以連線到同一個檔案進行共同編輯，不影響您硬碟中的其他個人檔案）。
    • 在「測試使用者 (Test users)」步驟中，務必新增您與家人的 Google 信箱，這樣大家才能順利登入！
  4. 左側選單選擇「憑證 (Credentials)」，點選上方「+ 建立憑證」>「OAuth 2.0 用戶端 ID」：
    • 應用程式類型選擇「網頁應用程式 (Web application)」。
    • 在 已授權的 JavaScript 來源 中新增兩個網址：
      🔹 本機測試：http://localhost:5000 或 http://127.0.0.1:5000 (若有在 local 測試)
      🔹 線上網址：https://shyshyhao.github.io
  5. 建立後，複製產生的 用戶端 ID (Client ID) 並貼到上方的輸入框中，點選下方「儲存 Client ID」。
  6. 👨‍👩‍👧‍👦 家庭協同編輯設定：
    當主要管理者 A 登入並成功同步後，雲端硬碟會自動產生一個 rental_helper_db.json 檔案。
    家人同步步驟：
    1. 主要管理者 A 登入 Google 雲端硬碟 (Google Drive) 網頁版，將該 rental_helper_db.json 共用（Share）給家人的 Google 帳號，並設定權限為「編輯者」。
    2. 家人 登入自己的雲端硬碟，務必先將自己硬碟中之前因測試重複建立的 rental_helper_db.json 檔案徹底刪除（並清理垃圾桶），確保硬碟中只留有 A 共享過來的檔案。
    3. 家人 在他們的手機或瀏覽器開啟此網頁，輸入相同的 Client ID。
    4. 家人 點選「中斷連線」以清除舊有暫存，接著點選「🔗 登入並啟用同步」並勾選「查看、編輯您的雲端硬碟所有檔案」權限登入。網頁即會自動偵測並連結到 A 共享的同一個資料庫檔案，實現多人即時同步編輯！

📱 單網頁應用程式 (SPA) 與本機優先

本工具為純前端單網頁應用程式 (Single Page Application, SPA)，無任何後端伺服器 (Serverless)。所有的資料預設均儲存於瀏覽器本機快取 (localStorage) 中。

📂 資料庫結構模式

資料在記憶體中以 state 全域物件進行管理，核心欄位包含：

  • state.rooms: 房間與房客資訊陣列（包含租金、上期欠繳、水費、獨立度數、繳款狀態等）。
  • state.settings: 收款帳戶、LINE 明細訊息模板，以及各樓層公電設定（state.settings.floor_elec）。
  • state.history: 歷年各月份結算歷史紀錄。

🔄 生命週期運作

當使用者異動任何資料時，會自動呼叫 saveToLocalStorage() 同步至本機。如果啟用了 Google Drive 同步，則會透過 GDrive API 上傳/下載 rental_helper_db.json 進行雲端雙向同步。

🔢 帳單計費核心公式

每位房客的應繳帳款小計是由下列幾項公式累加所得：
應繳小計 = 固定租金 + 獨立電費 + 樓層公共電費 + 固定水費 + 上期未繳(欠繳)

📋 各分項計算邏輯

  • 固定租金：讀取 room.rent (房客 1) 或 room.rent2 (房客 2)。年繳模式啟用時為 0 元。
  • 獨立電費：用電度數 (本期 - 上期) × 每度電費單價 (room.elec_price)。
  • 樓層公共電費：系統支援兩種模式：
    ✍️ 手動輸入：直接輸入本樓總公電費，由平分數除算之。
    ⚡ 度數計算：輸入上期及本期度數，依每度單價算出總金額，再除以平分份數。
    每個樓層的公共電費會由該樓層所有的主要/次要房客均攤。
  • 固定水費：由獨立欄位 room.water_fee 與 room.water_fee2 記錄，供各房間/房客儲存每月固定水費額。
  • 雙人房電費拆分：雙人房電費可設定「兩人平分 (50/50)」、「房客 1 全額」或「房客 2 全額」。

📅 月份結算 (Rollover) 流程

當點擊結算歸檔時，系統會執行以下結算自動化流程：
1. 複製並封存當月所有計算結果至 state.history。
2. 自動滾動電錶：將本期獨立電錶及各樓層公電錶的「本期度數」直接複製為下期「上次度數」，重設本期輸入格為空。
3. 自動累計欠繳：若勾選累計未繳，對於未付款的房客，會將應繳小計滾入下期「上期欠繳」中，已付款者下期欠繳歸零。

🔒 Google 帳密頻繁登入與密碼金鑰錯誤之謎

當在 iPhone (Safari) 開啟本程式，或每隔一段時間打開時，Google 常會彈出錯誤提示，宣稱「找不到您設備上的密碼金鑰 (Passkey)」，或需要您斷開重登。這是正常的瀏覽器第三方 Cookie 阻擋安全機制，其原理如下：

  1. 第三方 Cookie 阻擋原則： iOS Safari 及現代 Chrome 為了隱私，預設啟用了「防止跨網站追蹤」。當 Google OAuth SDK 載入 accounts.google.com 驗證登入狀態時，瀏覽器會阻擋其讀取 Google 的登入 Cookie，導致 Google 認為您正處於高風險不安全環境中。
  2. 安全挑戰機制： 當 Google 認定驗證 cookie 被阻擋、或是 implicit token (1小時過期) 過期需要靜默更新時，它會啟動 FIDO2 安全挑戰，強迫呼叫本機端設備的 Apple Passkey 安全驗證。而當設備未綁定對應的金鑰，就會跳出「找不到密碼金鑰」的惱人錯誤。

🛠️ 開發者推薦的最佳排除指引

當楊爸爸或家人使用遇到此問題，請依照以下步驟解決：

  • 方法 A (最推薦 - 改用 Chrome 瀏覽器)：在手機上下載 Chrome 瀏覽器開啟，相容性最穩定，最不易被 Google 彈出 Passkey 阻擋。
  • 方法 B (關閉跨網站追蹤阻擋)：若堅持使用 iPhone Safari，請前往手機的「設定」>「Safari」> 將「防止跨網站追蹤 (Prevent Cross-Site Tracking)」關閉，即可重獲流暢的靜默一鍵登入。
  • 方法 C (100% 穩定離線方案)：本軟體設計為本機離線優先。若在戶外完全無法登入 Google Drive，可以直接在系統設定點選「匯出資料備份」存入手機檔案中，下次點選「匯入備份還原」即可完美復原，不必依賴網路！

👨‍👩‍👧‍👦 雲端即時多人協同編輯

系統內建強大的雲端同步架構，完全利用 Google Drive 的安全共享權限：
1. 開發者 A (或楊爸爸) 登入 Google 並同步後，其雲端會建立 rental_helper_db.json 檔案。
2. 共享檔案：將該 json 檔案共享給家人的 Google 信箱。
3. 家人同步：家人打開網頁登入自己的 Google 信箱，系統會藉由 OAuth drive 權限自動掃描到被共享的 rental_helper_db.json 檔案，實現多人跨手機即時共用同一個資料庫！

🔗 離線同步連結機制 (LINE 多機同步)

在不使用 Google 同步的情況下，系統提供極具創意的 generateSyncLink()：
本功能會將當前的 state 狀態進行 gzip 壓縮與 Base64 加密，打包成一個長連結。
您只需要將此連結發送到家人的 LINE 群組中，家人在他們的手機上點擊該連結，網頁會自動解析連結中的 Base64 數據並覆蓋其 localStorage，瞬間完成多機一鍵同步！

👨‍👩‍👧‍👦 家人同步與連線教學

💡 親愛的家人，歡迎使用房租助手！
為了讓大家能像編輯同一份 Google Sheet 試算表一樣共用數據，請在第一次使用時，在您的手機或電腦上完成以下 4 個簡單連線步驟：
  1. 清除暫存與輸入 Client ID
    切換到右下角「系統設定」分頁。如果雲端同步狀態顯示已連線，請先點選 「🔌 中斷 Google 雲端連線」（以防連結到舊的測試檔案）。接著，確認 Client ID 欄位已填入管理者提供的編號，並點選「儲存 Client ID」。
  2. 點選「🔗 登入並啟用同步」
    點選按鈕後會彈出 Google 登入視窗，請輸入您自己的 Google 帳號和密碼進行登入。
  3. ⚠️ 重要：略過 Google 安全警告
    因為這個 App 是我們家庭獨立開發、未向 Google 申請昂貴的商業付費驗證，因此 Google 會顯示「Google 尚未驗證此應用程式」的警告。這是正常的！
    👉 請點選左下角的 「進階 (Advanced)」 ➡️ 再點選最下方的 「前往『楊爸爸房租助手』（不安全）」 即可安全繼續。
  4. ⚠️ 關鍵步驟：務必核取雲端硬碟權限
    登入的最後一步，Google 會詢問您是否授權此應用程式。
    👉 畫面上會出現一個「查看、編輯、建立及刪除您 Google 雲端硬碟中的所有檔案」核取方塊。請務必將它【打勾】（或點選「繼續/Continue」以允許所有要求的權限）！
    （註：只有核取此方塊，程式才能在您的雲端硬碟中搜尋並讀寫共用的 rental_helper_db.json 檔案，實現多人共用）。
🎉 完成後，您就可以與全家人隨時同步並共同編輯最新資料囉！
